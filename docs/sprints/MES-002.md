# Objective

Establish a single asynchronous programming model for the entire MiniConnect project.

This milestone introduces Kotlin Coroutines and StateFlow as the standard asynchronous architecture.

No Wi-Fi Direct features are added in this milestone.

The goal is to prepare the networking layer before implementing peer discovery.

---

# Background

Android's Wi-Fi Direct framework is callback-based.

MiniConnect should not expose Android callbacks outside the data layer.

Instead, Android callbacks will be converted internally into coroutine-friendly APIs.

This keeps the ViewModel and UI independent from Android framework implementation details.
